Stellar collisions

When two stars cross paths and end up colliding, they will expell material which will later interact with the radiation from the collision product. This interaction will lead to a brightness increase, which I wish to quantify by simulating different stellar collisions.

In order to predict these transients, I first need to simulate the collisions to obtain the amount of mass ejected. I will then use this mass as input into a Radiative Hydrodynamic code to study how the radiation emitted by the collision product will interact with the ejected matter.

For the first part of this project, I am modelling the collision with Phantom, a Smoothed Particle Hydrodynamics (SPH) code. SPH is a technique that discretises the hydrodynamic equations: it approximates a hydrodynamical system as a collection of particles (a more classical approach would be to simulate a fluid evolving in a grid). Each particle has its own properties (position, velocity, energy, etc) and one can interpolate the variables of interest (e.g. density) to visualise the behaviour of the fluid(s). While this process has many downsides, it is a good approximation in highly dynamical contexts, like stellar collisions, which can be computationally expensive in 3D grid-based Hydrodynamic codes.
